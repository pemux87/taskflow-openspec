## Why

TaskFlow MVP 구현 후 스토리보드 v2(42슬라이드)의 와이어프레임과 실제 구현된 UI 간에 다수의 시각적·기능적 불일치가 확인되었다. 설계 문서가 정의한 화면 상태, 에러 케이스, 인터랙션 패턴이 코드에 반영되지 않아 UX 일관성이 떨어지는 상태이다.

## What Changes

- **수정** 로그인 성공 시 `✓ 성공! 이동 중…` 상태 메시지 + 녹색 버튼 표시 (B·03)
- **수정** 회원가입 이메일 중복 에러(EMAIL_TAKEN)를 상단 배너 → 이메일 필드 인라인 표시 (B·02)
- **수정** 팀 생성 후 즉시 redirect 대신 초대코드 발급 화면 (초대코드 + 📋 복사 버튼 + 칸반 시작 버튼) 표시 (C·02)
- **수정** 초대코드 합류 후 즉시 redirect 대신 팀 미리보기 화면 (팀명 + 멤버 수 + 합류 버튼) 표시 (C·03)
- **수정** 칸반 빈 컬럼 empty state: TODO 컬럼에 📋 아이콘 + "첫 태스크 만들기" CTA, DOING/DONE은 "드래그로 이동" 안내 (D·02)
- **수정** 칸반 인라인 태스크 생성 폼에 담당자(assignee) 드롭다운 추가, 기본값 @me (D·03)
- **수정** 드래그 중 대상 컬럼 하이라이트 + "⬇ 여기에 놓기" 시각 피드백 (D·04)
- **수정** 태스크 모달에 생성자 이메일 + 생성 시각 메타 정보 표시 (D·05)
- **수정** 태스크 삭제 확인을 브라우저 `confirm()` → 커스텀 다이얼로그로 교체 (D·06)
- **신규** 비멤버 팀 접근 시 403 전용 화면 (🚫 + 안내 문구 + 내 팀으로 돌아가기) (D·07)
- **수정** 모바일 FAB `prompt()` → 슬라이드업 태스크 추가 모달 (F·01)
- **수정** 모바일 칸반 탭 클릭 전환에 좌우 스와이프 제스처 추가 (F·01)
- **수정** 채팅 폴링 실패 시 exponential backoff (5s→10s→20s→…→60s 상한) + 재시도 횟수 표시 (E·04)
- **수정** 모바일 채팅 본인 메시지 꾸욱 누르기 → 삭제 컨텍스트 메뉴 (F·02)
- **수정** 팀 멤버 패널에 가입 날짜(joined_at) 표시 (C·05)

## Capabilities

### New Capabilities
없음 (모두 기존 기능의 UI/UX 개선)

### Modified Capabilities
- `user-auth`: 로그인 성공 상태 표시, EMAIL_TAKEN 인라인 에러 위치 변경
- `team-management`: 팀 생성 후 초대코드 발급 화면, 합류 후 팀 미리보기 화면, 멤버 패널 joined_at 표시
- `kanban-board`: empty state, 인라인 담당자, 드래그 시각화, 모달 메타 정보, 커스텀 삭제 확인, 403 화면, FAB 모달, 모바일 스와이프
- `team-chat`: exponential backoff 폴링, 모바일 꾸욱 누르기 삭제

## Impact

- **프론트엔드**: `public/js/views/` 하위 5개 파일 수정 (login.js, signup.js, team.js, kanban.js, chat.js)
- **백엔드**: 변경 없음 (기존 API 응답 구조 그대로 활용)
- **의존성**: 추가 라이브러리 없음
